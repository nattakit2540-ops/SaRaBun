# AGENTS.md

แนวทางสำหรับ agent ที่พัฒนาโปรเจกต์ E-SARABAN SCHOOL

## เป้าหมายระบบ

ระบบสารบรรณโรงเรียนสำหรับโรงเรียนชุมชนบ้านหนองผึ้ง (ประพันธ์คุรุราษฎร์อุทิศ) รองรับคำสั่งโรงเรียน บันทึกข้อความ หนังสือส่ง เลขทะเบียนตามปี พ.ศ. และการบันทึกข้อมูลจริงลง Firebase

## หลักการสำคัญ

- ผู้ใช้งานทั่วไปต้องเข้าระบบและลงทะเบียนเลขเอกสารได้ทันทีโดยไม่ต้อง Login
- เมื่อมี Firebase config ให้ใช้ Anonymous Login เบื้องหลังสำหรับผู้ใช้งานทั่วไป
- การแก้ไขหรือลบเอกสารต้องทำได้เฉพาะ role `admin` และควรถูกบังคับซ้ำด้วย Firebase custom claim `admin`
- หน้าแรกต้องมีปุ่ม `ใช้งานทั่วไป` สำหรับเข้าใช้งานทันที และปุ่ม `Admin` สำหรับแสดงฟอร์ม Login
- Admin local ใช้ `stamp45240 / Punbnk48` และไม่แสดงคำแนะนำรหัสผ่านบนหน้าแรก
- Demo Mode เดิมยังใช้ localStorage และไม่บันทึก Firebase
- Admin/ธุรการใช้งานจริงด้วย Firebase Email + Password
- เลขเอกสารที่กรอกย้อนหลังต้อง sync counter เพื่อให้เลขอัตโนมัติรันต่อ
- ห้ามลดทอนข้อมูล audit log เมื่อแก้ไขสถานะ แก้ไขเอกสาร รับหนังสือ หรือนำเข้าเอกสาร

## ไฟล์สำคัญ

- `index.html` โครงหน้าเว็บและจุดโหลด script
- `css/style.css` ธีม Colorful Minimal และฟอนต์ Itim
- `js/app.js` logic หลักทั้งหมดของเว็บแบบเปิดจากไฟล์ได้
- `js/firebase-config.js` Firebase config และค่าเริ่มต้นของระบบ
- `firebase.json`, `firestore.rules`, `storage.rules` สำหรับ deploy Firebase

## การทดสอบก่อนส่งงาน

- เปิด `index.html` ต้องเห็นปุ่ม `ใช้งานทั่วไป` และเข้า Dashboard ได้โดยไม่ต้องกรอกรหัส
- สร้างเอกสารทั้ง 3 โมดูลได้
- กรอกเลขย้อนหลังแล้วเลขถัดไปต้องรันต่อ
- ผู้ใช้ public ต้องเปิดระบบได้ทันทีโดยไม่ต้อง Login และต้องไม่เห็นปุ่มแก้ไข/ลบ
- Login admin ด้วย `stamp45240 / Punbnk48` แล้วต้องเห็นปุ่มแก้ไข/ลบและเมนูตั้งค่าระบบ
- ตรวจ `node --check` กับ `js/app.js` และ `js/firebase-config.js`
